# STM32H563 UART Configurable Console

Interactive serial console on the NUCLEO-H563ZI that lets you reconfigure the active UART peripheral at runtime via typed commands — no reflash required.

## What it does

The firmware starts a `>` prompt on `USART3` at `115200 8N1` through the ST-LINK virtual COM port. Three commands are supported:

```
> help
Commands:
  help
  uart show
  uart set <baud> <8n1|8e1|8o1|8n2>

> uart show
UART console: 115200 8N1

> uart set 9600 8n1
Applying UART console: 9600 8N1. Update terminal now.
>
```

After `uart set`, the firmware drains pending TX data, deinitializes `USART3`, rebuilds it with the new parameters, and restarts RX interrupt reception. The host terminal must be updated to match.

Accepted baud range: `1200` – `1000000`.  
Supported formats: `8n1`, `8e1`, `8o1`, `8n2`.

The board also continues blinking `LD2` and responding to the `B1` user button throughout.

## How it works

All I/O is non-blocking — no `HAL_UART_Transmit` blocking calls in the application path.

**TX:** A 16-slot circular queue holds outgoing strings. `LogUart()` appends a message; if the UART is idle, `HAL_UART_Transmit_IT()` starts the first transfer. The TX-complete callback advances the queue and starts the next message.

**RX:** `HAL_UART_Receive_IT()` captures one byte at a time into a 32-byte circular FIFO. The ISR stays short — push the byte and restart reception immediately. The main loop pops bytes, echoes them, builds a command line, and dispatches it on Enter.

**Reconfiguration:** Before changing UART settings, the firmware waits for the TX queue to drain, aborts RX, deinitializes `USART3`, updates `huart3.Init`, reinitializes, and restores the STM32H5 FIFO threshold settings (required after every `HAL_UART_Init` on H5 — they are not preserved by the HAL reinit path).

## Hardware

| Item | Value |
|------|-------|
| Board | NUCLEO-H563ZI |
| MCU | STM32H563ZIT6 (ARM Cortex-M33, 250 MHz) |
| Console UART | USART3 via ST-LINK VCP |
| Default settings | 115200 8N1 |

## Build & Flash

1. Open STM32CubeIDE.
2. **File → Import → Existing Projects into Workspace** — select this directory.
3. **Project → Build All**.
4. **Run → Debug** (or Run) to flash via ST-LINK.
5. Open a serial terminal at `115200 8N1` on the ST-LINK VCP port.
6. Reset the board — the `>` prompt appears.
